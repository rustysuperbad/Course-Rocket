# CourseRocket

An AI-powered learning platform that turns any topic into a structured, trackable course with lessons, videos, research papers, and module quizzes.

## What It Does

CourseRocket helps learners go from "I want to learn this" to a complete guided learning path in minutes:

- Generates a full multi-module course outline from a topic prompt
- Writes lesson content for each module
- Curates relevant YouTube videos per module
- Fetches research papers for deeper reading
- Adds module quizzes with answer explanations
- Tracks progress and quiz scores over time

## Demo Flow

1. Enter a topic on the landing page
2. Sign in with Google or email/password
3. Generate a personalized course
4. Study module lessons, videos, and papers
5. Take quizzes and track completion progress

## Tech Stack

- **Frontend:** React + Vite
- **Routing:** React Router
- **Auth + DB:** Firebase Auth + Firestore
- **AI generation:** Groq Chat Completions API
- **Content sources:** YouTube Data API + OpenAlex
- **Animations/UI:** Framer Motion + custom styled components

## Local Setup

### 1) Clone and install

```bash
git clone <your-repo-url>
cd courserocket
npm install
```

### 2) Configure environment variables

Create a `.env` file in the project root with:

```env
VITE_GROQ_API_KEY=your_groq_api_key
VITE_YOUTUBE_API_KEY=your_youtube_data_api_key
```

### 3) Start development server

```bash
npm run dev
```

Then open the local URL shown by Vite (usually `http://localhost:5173`).

## Firebase Setup Notes

This app uses:

- Firebase Authentication (Google + Email/Password)
- Firestore database (`courses` collection)

For production use, make sure your Firebase Authentication methods and Firestore rules are configured correctly.

## Data Model (Firestore)

Each generated course is saved in the `courses` collection, with fields similar to:

- `userId`
- `topic`
- `title`
- `description`
- `level`
- `estimatedHours`
- `modules` (lessons, videos, papers, quiz)
- `progress` (module completion + quiz score)
- `createdAt`

## How Course Generation Works

When a user requests a topic:

1. Course structure is generated by Groq (JSON schema based)
2. Lessons are generated in chunks for module groups
3. YouTube videos are fetched from the YouTube Data API
4. Research papers are fetched from OpenAlex
5. Results are merged and saved to Firestore

The generator includes retry/fallback behavior for rate limits and malformed JSON responses.

## Security and Production Recommendations

- Move Firebase config values in `src/lib/firebase.js` to environment variables
- Add stricter Firestore security rules before public deployment
- Restrict API keys by domain and service usage in provider consoles
- Add server-side proxy/endpoints for third-party APIs if you need stronger key protection

## Troubleshooting

- **Missing Groq key:** ensure `VITE_GROQ_API_KEY` exists in `.env`
- **No YouTube results:** verify `VITE_YOUTUBE_API_KEY` and API quota
- **Auth popup blocked:** allow popups for your localhost domain
- **Empty course or partial content:** retry generation (LLM/API rate limits can occur)
